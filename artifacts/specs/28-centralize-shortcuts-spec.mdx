## Context

Promoted from approved frame: `artifacts/frames/28-centralize-shortcuts-frame.mdx`.

Hotkey labels in `overlay.py:259` are hardcoded and stale (`A+Sh+Sp` etc.), while the AHK script, docs, and example config already use the correct values (`ctrl+space`, `alt+shift+esc`, `alt+shift+tab`). This spec defines how a single source of truth in `[stt]` config propagates to overlay labels at spawn time.

## Goal

Overlay badge labels always reflect the actual hotkeys from `voicecli.toml [stt]`, with `config.py` as the single source of defaults.

## Users

- **Primary:** Any user running the dictate daemon — they see correct hotkey labels in the overlay without manual sync.
- **Secondary:** Anyone customizing hotkeys in `voicecli.toml` — their custom values appear in the overlay automatically after daemon restart.

## Expected Behavior

1. User runs `voicecli dictate` (starts daemon, which spawns overlay).
2. Daemon calls `load_stt_config()` → gets `hotkey`, `hotkey_cancel`, `hotkey_mode` (from toml or defaults).
3. Daemon spawns overlay process, passing hotkeys as env vars: `VOICECLI_OVERLAY_HOTKEY_TOGGLE`, `VOICECLI_OVERLAY_HOTKEY_CANCEL`, `VOICECLI_OVERLAY_HOTKEY_MODE`.
4. `WaveformOverlay.__init__()` reads the env vars, converts each to a short display string, and uses them as badge labels (Stop / Cancel / Mode).
5. With default config: Stop badge = `Ctrl+Sp`, Cancel badge = `A+Sh+Esc`, Mode badge = `A+Sh+Tab`.
6. User with custom `hotkey = "alt+f4"` in `voicecli.toml` sees `A+F4` in the Stop badge after daemon restart.

### Hotkey display format conversion

`_hotkey_badge(hotkey_str: str) -> str` — pure function, lives in `overlay.py`:

| Config value | Badge |
|-------------|-------|
| `ctrl+space` | `Ctrl+Sp` |
| `alt+shift+esc` | `A+Sh+Esc` |
| `alt+shift+tab` | `A+Sh+Tab` |
| `ctrl+shift+r` | `C+Sh+R` |

Abbreviation rules (applied left-to-right across `+`-split parts):
- `ctrl` → `Ctrl` when it is the only modifier (e.g. `ctrl+space`); `C` when combined with other modifiers (e.g. `ctrl+shift+r`)
- `alt` → `A`
- `shift` → `Sh`
- `space` → `Sp`
- `esc` → `Esc`
- `tab` → `Tab`
- other → capitalize first letter; unknown parts pass through gracefully (no crash, no empty badge)

## Breadboard

| Affordance | Handler | Data |
|------------|---------|------|
| U1: Config `[stt]` in toml | `config.load_stt_config()` | Returns `SttConfig` dict with 3 hotkey keys + defaults. Both `_KNOWN_STT` (for type coercion) and the `result` seed dict must be updated — two mutation points. |
| N1: `_spawn_overlay(mode, hotkeys)` | `stt_daemon._spawn_overlay()` — signature becomes `(mode, hotkey, hotkey_cancel, hotkey_mode)` or takes `cfg: dict`; single call site at line 690 | Sets 3 `VOICECLI_OVERLAY_HOTKEY_*` env vars from `SttConfig` |
| N2: Badge label rendering | `overlay._hotkey_badge(s)` | Pure converter: "ctrl+space" → "Ctrl+Sp" |
| N3: `WaveformOverlay.__init__()` | Reads env vars → calls `_hotkey_badge()` | Replaces hardcoded list at line 259 |
| S1: Default fallback | `load_stt_config()` default dict | If toml absent or keys missing → hardcoded defaults in `_KNOWN_STT` |

**Wiring:**
- U1 → N1: daemon reads config once at spawn, passes all 3 hotkeys
- N1 → N3: env vars bridge daemon → overlay process boundary
- N3 → N2: overlay calls formatter once per badge at init time
- S1 → N1: defaults flow through same path; no special case in overlay
